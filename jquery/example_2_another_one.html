#2: A second example

This is exactly the same form as #1 with exactly the same validation rules. The only change is that it uses the display-html value for the displayType configuration option, with a single hidden <div> tag in the page, used as a placeholder to target the errors. Also, it defines a custom error class (.errorFieldDemo2) for the error message fields - making them appear with a yellow background and red text.
